# drafts/

Scratch space for new material that hasn't been reviewed yet.

Workflow:

1. New material, a lecture draft, a write-up, an exercise sketch, goes here, with a status comment at the top: `<!-- status: unreviewed | last-reviewed: never -->`.
2. Someone reads it end to end, fixes the errors, checks the citations and the code.
3. Once it holds up, it moves to wherever it belongs (`notes/`, `exercises/`, ...) with the status set to `reviewed` and a real `last-reviewed` date.

Nothing in here is trusted. Don't link to it from `notes/` or cite it as if it's settled. If a draft has been sitting here untouched for a long time, either review it or delete it: stale drafts pretending to be content are the thing this directory exists to prevent.
